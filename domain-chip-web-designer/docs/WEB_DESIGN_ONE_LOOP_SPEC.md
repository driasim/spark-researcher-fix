# Web Design One Loop Spec

This domain chip is built to run a disciplined learning loop for website and dashboard design.

## Objective

Increase `web_design_score` without letting accessibility, task clarity, or conversion collapse.

## One Loop

1. Ingest reference material from award ecosystems and durable UI/UX sources.
2. Normalize findings into reusable mechanisms:
   visual direction, narrative architecture, interaction model, proof surface, conversion path, and accessibility posture.
3. Generate candidate design systems instead of single screenshots or isolated flourishes.
4. Evaluate each candidate with the benchmark:
   `award_signal_score`, `ux_system_score`, `interaction_craft_score`, `conversion_clarity_score`, `accessibility_safety_score`.
5. Packetize lessons only when they are specific, bounded, and non-vanity.
6. Promote candidates into doctrine only after benchmark strength and real-world checks.

## Core Rule

Award resemblance is inspiration, not proof.

A candidate can look premium and still fail if:

- the first screen is confusing
- motion obscures hierarchy
- the CTA path is weak
- proof is generic
- the dashboard hides primary tasks
- accessibility is treated as optional polish

## Comparison Classes

- `exploratory_frontier`: interesting direction, not yet trustworthy
- `heuristic_frontier`: coherent and worth probing, still synthetic
- `benchmark_grounded`: survived deterministic benchmark thresholds
- `realworld_validated`: survived live usability or product outcomes

## Mutation Axes

- `site_type`
- `design_direction`
- `narrative_model`
- `interaction_model`
- `proof_surface`
- `conversion_strategy`
- `accessibility_mode`

## Anti-Patterns

- design tourism: copying award surfaces without understanding why they work
- motion theater: adding animation that weakens information hierarchy
- dashboard cosplay: making dashboards look advanced while slowing core tasks
- taste laundering: calling something "premium" when it is just sparse
- accessibility debt hiding inside "experimental" creative direction
