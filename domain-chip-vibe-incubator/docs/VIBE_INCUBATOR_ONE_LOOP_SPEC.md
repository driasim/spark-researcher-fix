# Vibe Vibe One-Loop Spec

Use this chip when the incubator needs one governing loop instead of disconnected startup picking, content posting, and product building motions.

## Goal

Turn a solo operator plus agentic workflows into a compounding incubator that:

- selects better venture plays
- validates them faster
- reuses build assets
- captures doctrine from launches and failures
- does not drift into startup theater

## Governing Loop

Each pass should:

1. refresh current incubator state
2. classify the main bottleneck
3. test or suggest the next bounded venture play
4. update packets and watchtower
5. decide whether any play deserves real-world validation

## Bottleneck Labels

- `model_gap`
- `distribution_gap`
- `automation_gap`
- `learning_gap`
- `revenue_gap`
- `resilience_gap`

## Routing

- `model_gap` -> shrink the venture surface
- `distribution_gap` -> change the distribution engine before adding product scope
- `automation_gap` -> move repeated work into templates, agents, or internal tools
- `learning_gap` -> strengthen post-launch review and reusable asset capture
- `revenue_gap` -> push paid validation harder
- `resilience_gap` -> tighten trust, auditability, and human review before scaling

## Rule

Do not let the incubator open more venture lanes than the current operator system can validate, review, and learn from.
