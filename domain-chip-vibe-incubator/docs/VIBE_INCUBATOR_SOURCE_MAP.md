# Vibe Incubator Source Map

This chip should learn from source families that improve real incubator judgment rather than generic startup residue.

## High-Signal Source Families

- venture studio and incubator operator writeups
- founder operating memos with concrete launch details
- postmortems on failed startup operating systems
- essays and case studies on agentic workflow design
- security and resilience lessons for small teams running leveraged systems
- distribution case studies for solo operators and tiny teams

## Low-Signal Families

- generic startup motivation threads
- vanity funding narratives without operating detail
- hype about AI agents with no workflow economics
- growth stories that hide team size or leverage source

## Research Rule

Do not let broad startup content become incubator doctrine directly.
Source notes should become bounded packet candidates first, then doctrine only after fixed-evaluator and real-world support improve.
