# Agentic Marketing One-Loop Spec

This chip governs one startup growth loop:

1. refresh research state
2. identify the bottleneck
3. test the next coherent system candidate
4. packetize the result
5. update watchtower
6. decide whether promotion or real-world validation is justified

## Mission

Build a startup distribution system that can be run by a small team through:

- workflow automations
- LLM-assisted research and packaging
- agentic repurposing
- measurable capture surfaces
- CRM and analytics closure

The target is not "more marketing."
The target is a closed-loop growth operating system.

## Bottlenecks

Use one label only:

- `knowledge_gap`
- `trial_gap`
- `ranking_gap`
- `promotion_gap`
- `coverage_gap`
- `contradiction_gap`

Interpretation in this domain:

- `knowledge_gap`: weak ICP, pain, or offer clarity
- `trial_gap`: not enough coherent motion-channel-offer systems tested
- `ranking_gap`: enough tests, but the evaluator cannot separate winners from noise
- `promotion_gap`: benchmark winners exist but live-pilot rules are not defined
- `coverage_gap`: too much dependence on one channel or one tool surface
- `contradiction_gap`: a previously trusted system is no longer holding up

## Promotion Gate

Promotion from benchmark scaffold to live pilot requires:

- `distribution_system_score >= 0.80`
- `automation_coverage_score >= 0.68`
- `feedback_closure_score >= 0.66`
- explicit claim, mechanism, boundary, and next probe

## Domain Rule

This chip treats startup marketing as a systems problem:

- research should create better offers
- offers should create better assets
- assets should create attributable capture
- capture should create better CRM state
- CRM and product data should improve the next iteration

If any of those edges are broken, the system is not ready for doctrine promotion.
