# Agentic Marketing Source Map

This chip treats startup growth as a composable system, not a single tool choice.

## System Layers

### Research And Signal Capture

- product analytics
- CRM events
- search demand
- community pain signals
- support conversations
- surveys and direct user feedback

### Packaging And Asset Production

- offer framing
- proof extraction
- content repurposing
- page generation
- outbound sequence generation
- attribution tagging

### Routing And Capture

- chatbot or conversational intake
- inbox handoff
- landing or comparison pages
- calendar routing
- CRM sync

### Feedback Closure

- analytics
- CRM pipeline outcome
- survey responses
- support transcripts
- operator review notes

## Open-Source Tool Roles

- `n8n`: workflow orchestration and agent routing
- `PostHog`: analytics, experiments, and product usage traces
- `Twenty`: CRM and account memory
- `Mautic`: lifecycle campaigns and outbound automation
- `Chatwoot`: inbox and human handoff
- `Typebot`: conversational capture
- `Formbricks`: survey-based voice-of-customer
- `Dub`: short-link routing and attribution

## Discovery Surfaces

- `Trendshift`: use as a discovery index for rising open-source repos worth triaging for the stack
- GitHub repository pages: use as the primary repo-level truth surface once a candidate is discovered
- official docs: use for product semantics, setup patterns, and capability confirmation

## Evidence Rule

Official repos and docs belong in the approved source set.
Community examples can be discovery seeds, but they should not become doctrine without benchmark or live evidence.
